# 사설 발급기관 개인 키(key) 파일 생성

입력

openssl genrsa -out cacert.key 2048

# 인증서 서명 요청 파일(csr) 생성

구분 설명 예제 값
Country Name 국가 코드 KR
State or Province Name 시/도의 전체 이름 Seoul
Locality Name 시/군/구 등의 이름 Gangnam-gu
Organization 회사 이름 Example
Organization Unit 부서명 Sales
Common Name SSL 인증서를 설치할 서버의 FQDN *.example.org

입력

openssl req -new -key cacert.key -out cacert.csr

결과

You are about to be asked to enter information that will be incorporated
into your certificate request.
What you are about to enter is what is called a Distinguished Name or a DN.
There are quite a few fields but you can leave some blank
For some fields there will be a default value,
If you enter '.', the field will be left blank.
-----

State or Province Name (full name) []:Seoul
Locality Name (eg, city) [Default City]:Gangnam-gu
Organization Name (eg, company) [Default Company Ltd]:Example
Organizational Unit Name (eg, section) []:Sales
Common Name (eg, your name or your server's hostname) []:*.example.org
Email Address []:cloud.rnd@samsung.com

Please enter the following 'extra' attributes
to be sent with your certificate request
A challenge password []:

# 사설 인증서 파일(crt) 생성

입력

openssl req -x509 -days 365 -key cacert.key -in cacert.csr -out cacert.crt -days 365
